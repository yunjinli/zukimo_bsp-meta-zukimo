Example applications

As mentioned in Introduction to Radar, Radar is useful in a number of applications. Some of these applications are discussed briefly in these examples.

Presence Detection APP

Presence Detection is an application of a radar system, where the radar is able to identify whether there are any targets in a specified vicinity. Specifically, the radar continuously detects targets within an angle coverage up to a certain distance. The maximum detection and minimum detection range are parameters that may be configured through the radar settings. Besides these, other parameters of the radar may also be configured. Presence detection can be further utilized for applications such as keyword-less authentication, automatic interaction of smart devices with the user and the advantage of using a radar system is that it can operate 24x7 unmanned without getting affected by lighting conditions in the room.

The following figure shows a high-level representation of presence detection:

Illustration of presence detection


Infineon's Presence Detection system, described here, uses an FMCW radar. A detailed overview of the FMCW radar system is given in Operation of FMCW based system. This algorithm consists of the following components:

  • State Machine
  • Absence State Processing
  • Presence State Processing

The state processing has been released by using various modules from the Infineon radar SDK library, for more details please refer to sections Absence State Processing and Presence State Processing.

The application also offers a set of configuration parameters in form of a .json file to:

  • Configure the radar device based on some required presence detection metrics
  • Configure presence detection algorithm for tweaking it in a desired environment.

If there is no .json file specified, these configuration parameters assume some pre-defined default values at which a presence sensing system has been tested.

Installation procedure

The application source files are located in the apps/c/presence_detection folder. To install the project, follow the procedure in Building SDK from source code.

Operation of FMCW based system

Target estimation from FMCW

The figure shows the basic operation of FMCW radar system. The transmitted signal is a frequency modulated signal. The received signal at the receiver output is obtained after mixing the transmitted signal and the received signal. This generates an IF signal which can be seen in the figure as a sinusoidal signal. The frequency of the IF signal corresponds to the beat frequency. This beat frequency is used to estimate the range of the target. In case of multiple targets, this IF signal comprises of sinusoids with different frequencies corresponding to the target ranges. The figure shows how one can obtain the beat frequency by computing a Fast Fourier Transform over the IF signal. The Range of the target can be estimated from the beat frequency as follows:

\[ R = \frac{c T_c f_b}{2 B} \]

Basic principle of operation of an FMCW radar system
Doppler estimation with FMCW

Motion in targets produces changes in the IF signal at the receiver. The frequency of the IF signal remains almost the same (for slow moving targets), however the phase of the IF signal shows changes. This change in phase of the IF signal from one chirp to another is related to the radial velocity of the targets and can be given as:

\[ \delta\omega = \frac{4 \pi v_r T_c}{\lambda} \]

The change of phase can be evaluated over multiple chirps to obtain the velocity of the moving target. This can be accomplished by computing FFT over the chirps.

Multiple Chirp configuration of FMCW for Doppler Estimation

State Machine

A state machine is a mathematical model of computation and can be exactly in one of the finite number of states at a given time. The Presence Detection algorithm operates as a simple state machine with 2 states namely: Presence and Absence. The following figure depicts an overview of the state machine:

Presence Detection state machine

Depending on the current state (Presence or Absence), the corresponding processing algorithm is active. The processing algorithms act successively on each "frame" of radar data arriving from the FMCW radar device. The frame consists of IF data coming from a set of chirps, as described in Operation of FMCW based system.

Absence State Processing

The absence state processing is depicted by the following block diagram:

Absence State Processing

The Absence state processing depicted in the figure happens once every "frame" of raw data (IF data) is obtained from the radar device. The blocks in the diagram with bold letters correspond to modules from the radar_sdk library. These modules are implemented in an object oriented fashion in C, and so they are instantiated handles to structures. Details about each of these modules are as described below.

  • A handle for an instance of Range Spectrum module ifx_RS_t, operating in the coherent integration mode (see ifx_RS_Mode_t) performs the range resolution on the incoming frame data, gives out a range spectrum.
  • A handle for an instance of MTI module ifx_MTI_t with its alpha parameter set to 1 suppresses static targets from the range spectrum coming out of the ifx_RS_t
  • A handle for an instance of Peak Search module ifx_Peak_Search_t named "absence_peak_handle" identifies peaks (targets) higher than a certain adaptive threshold setting from the resultant range spectrum. These peaks are used as one of the deciding factors for presence detection, as shown in the absence state processing figure. This "absence_peak_handle" module is instantiated with a ifx_Peak_Search_Config_t having the following values (by default):
    • value_per_bin = calculated from chirp bandwidth, and number of samples per chirp
    • search zone start = IFX_MIN_DETECTION_RANGE_M
    • search zone end = IFX_MAX_DETECTION_RANGE_M - (IFX_MAX_DETECTION_RANGE_M * IFX_RANGE_HYSTERESIS / 100)
    • threshold factor = 3
    • threshold offset = 0
    • max number of peaks = 5
  • An instance of ifx_Peak_Search_t named "absence_fine_peak_handle" identifies peaks higher than a certain adaptive threshold setting from the resultant range spectrum. The threshold factor for this peak search module is lesser than that of "absence_peak_handle", so it detects range bin indices of much more subtle targets ("fine peaks")) in the search zone. These range bin indices of fine peaks are used to calculate the second "slow time" FFTs on the range spectrum matrix in the doppler domain, as shown in the absence state processing figure. This "absence_fine_peak_handle" module is instantiated with a ifx_Peak_Search_Config_t having the following values:
    • value_per_bin = calculated from chirp bandwidth, and number of samples per chirp
    • search zone start = IFX_MIN_DETECTION_RANGE_M
    • search zone end = IFX_MAX_DETECTION_RANGE_M
    • threshold factor = 1.2
    • threshold offset = 0
    • max number of peaks = 5
Note
It may be noticed that this is similar to the range Doppler processing described in Range Doppler . However, the difference here is that the complete range Doppler map is NOT being calculated by the presence sensing algorithm but just of a maximum of 5 range bins corresponding to identified "fine_peak" targets (by the "absence_fine_peak_handle" ifx_Peak_Search_t module) from the range spectrum matrix.

The slow time FFT or Doppler FFT spectrum are also analyzed for peaks, with a fixed Doppler threshold value (hard-coded to 0.0002) and in fixed doppler bins to detect only targets moving towards the radar. The decision whether a target is present or not is made based on whether any peaks are obtained from the "absence_peak_handle" module and whether any peaks are obtained from the Doppler FFTs ("slow time FFTs") higher than the Doppler threshold value. If these conditions are satisfied for a fixed number of consecutive frames then that decides a definitive presence detection and the state change to Presence occurs, otherwise the state remains as Absence.

Presence State Processing

The presence state processing is depicted by the following block diagram:

Presence State Processing

The Presence state processing depicted in the figure happens once every "frame" of raw data (IF data) is obtained from the radar device. The blocks in the diagram with bold letters correspond to modules from the radar_sdk library. These modules are implemented in an object oriented fashion in C, and so they are instantiated handles to structures. Details about each of these modules are as described below.

  • A handle for an instance of Range Spectrum module ifx_RS_t , operating in the coherent integration mode (see ifx_RS_Mode_t) performs the range resolution on the incoming frame data, gives out a range spectrum.
  • A handle for an instance of MTI module ifx_MTI_t with its alpha parameter set to 1 suppresses static targets from the range spectrum coming out of the ifx_RS_t
  • A handle for an instance of Peak Search module ifx_Peak_Search_t named "presence_peak_handle" identifies peaks (targets) higher than a certain adaptive threshold setting from the resultant range spectrum. These peaks are used as one of the deciding factors to continue presence detection, as shown in the presence state processing figure. This "presence_peak_handle" module is instantiated with a ifx_Peak_Search_Config_t having the following values (by default):
    • value_per_bin = calculated from chirp bandwidth, and number of samples per chirp
    • search zone start = IFX_MIN_DETECTION_RANGE_M
    • search zone end = IFX_MAX_DETECTION_RANGE_M + (IFX_MAX_DETECTION_RANGE_M * IFX_RANGE_HYSTERESIS / 100)
    • threshold factor = 2.5
    • threshold offset = 0
    • max number of peaks = 1

The decision whether a target is present or not is made based on whether any peaks are obtained from the "presence_peak_handle" module. If no peaks are detected by the presence peak module for a fixed number of consecutive frames then that decides a definitive absence of targets and the state change to Absence occurs, otherwise the state remains in Presence.

The presence sensing configuration parameters are typically quantities that can tweak the algorithm that acts on the frame data coming from the device. The tweaking can be done to adjust the performance of the algorithm in specific environments.

If there is no .json file specified, the default parameters are applied. These are defined in presence_sensing_defaults.h.

Segmentation APP

Segmentation is a command-line application of a radar system, which is based on the Presence Detection APP and demonstrates the Segmentation algorithm provided by the SDK.

The Segmentation APP implements a "yes-or-no" presence detection in up to 3 segments, which are configurable up to 120 degrees. Presence can be shown at one or more segments at the same time, within a configurable angular field of view (15 degrees step), and a configurable distance (up to 5m). The minimum supported distance is 1m, below which the results are undefined.

Installation procedure

The application source files are located in the apps/c/segmentation folder. To install the project follow the procedure in Building SDK from source code.

Radar device configuration

Device specific configuration parameters are described in the Device configuration guidelines.

Segmentation antenna configuration

The Segmentation APP considers only 2 Rx antennas regardless of the number of Rx antennas in device configuration - it selects the antennas based on given orientation.

Segmentation algorithm configuration

The segmentation configuration parameters are typically quantities that can tweak the algorithm that acts on the frame data coming from the device. The tweaking can be done to adjust the performance of the algorithm in specific environments.

For convenience, the application can be configured through a .json file. See Json Configuration.

If there is no .json file specified, the default parameters are applied. The default mode for segmentation is IFX_SEGMENTATION_500MHZ_LANDSCAPE defined in ifx_Segmentation_Mode_t .

Raw Data APP

Raw Data is a very simple example application that illustrates how to configure the radar sensor and fetch raw data. Based on the default Radar device configuration parameters, it configures the radar sensor and sets up a function to process the resulting time-data measurement frames.

The function process_frame() iterates through the raw data, showing a possible way of processing the antenna signal. The goal in this application is to sum up all chirps into one vector.

Installation procedure

The application source files are located in the apps/c/raw_data folder. To install the project follow the procedure in Building SDK from source code.

Constant Wave APP

Constant Wave is a very simple example application that illustrates how to configure the radar sensor in Constant Wave mode and capture frames of raw data. Based on the default Radar device configuration parameters, it configures the radar sensor and sets up a function to process the resulting time-data measurement frames.

The function process_frame() in a loop catches NUM_FETCHED_FRAMES(5) frames in Constant Wave mode and iterates through the raw data, showing a possible way of processing the antenna signal.

Installation procedure

The application source files are located in the apps/c/constant_wave folder. To install the project follow the procedure in Building SDK from source code.
